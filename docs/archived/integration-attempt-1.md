# Integration Attempt 1 â€” Git + NextGlk

## Purpose

Analyse the minimum changes required to make the Git VM link against NextGlk
instead of CheapGlk. No Git source code is modified. No NextGlk implementation
code is written. This document records the gaps that must be filled.

---

## 1. How Git Currently Links Against CheapGlk

### Build mechanism

`thirdparty/git/Makefile` lines 8â€“14:

```makefile
GLK = cheapglk
GLKINCLUDEDIR = ../$(GLK)        # â†’ ../cheapglk
GLKLIBDIR = ../$(GLK)            # â†’ ../cheapglk
GLKMAKEFILE = Make.$(GLK)       # â†’ Make.cheapglk
```

Line 43:

```makefile
include $(GLKINCLUDEDIR)/$(GLKMAKEFILE)   # includes ../cheapglk/Make.cheapglk
```

`Make.cheapglk` (generated by CheapGlk's Makefile) defines:

```makefile
LINKLIBS =
GLKLIB = -lcheapglk
```

Line 47:

```makefile
LIBS = -L$(GLKLIBDIR) $(GLKLIB) $(LINKLIBS) -lm
```

Result: **`git` links with `-L../cheapglk -lcheapglk -lm`**

### Startup path

`git_unix.c` includes `<glkstart.h>` from the Glk library directory and uses:

| Symbol | Usage |
|--------|-------|
| `glkunix_argumentlist_t` | Type of `glkunix_arguments[]` |
| `glkunix_arguments[]` | Defines command-line argument format |
| `glkunix_startup_code()` | Called by Glk library before `glk_main()` |
| `glk_main()` | Defined by the **program** (Git), called by Glk library |
| `glkunix_stream_open_pathname()` | Opens game file as Glk stream (non-MMAP path) |

### Header dependency

`git_unix.c` uses `#include <glk.h>` and `#include <glkstart.h>`, both found via
`-I$(GLKINCLUDEDIR)` (`-I../cheapglk`). `glkop.c` uses `#include "glk.h"` and
`#include "gi_dispa.h"`.

## 2. Source Files Expected from a Glk Implementation

The Git Makefile expects the Glk library to provide:

### Build artefacts

| What | Details |
|------|---------|
| `Make.$(GLK)` | Makefile snippet defining `GLKLIB` and `LINKLIBS` |
| `lib$(GLK).a` | Static library (e.g. `libcheapglk.a`) |
| `glk.h` | Public Glk API (NextGlk has this) |
| `gi_dispa.h` | Dispatch-layer types (NextGlk has this) |
| `gi_dispa.c` | Dispatch-layer routing (NextGlk has this) |
| `glkstart.h` | Unix startup interface (NextGlk does **not** have this) |
| `glkstart.c` | Unix startup implementation (NextGlk does **not** have this) |

### Glk API functions

All functions declared in `glk.h` (both mandatory and `#ifdef`-guarded optional
modules) must resolve at link time. `gi_dispa.c`'s `function_table` and
`gidispatch_call()` reference every function that the dispatch layer can route,
so even stubs must exist for every function in that table.

## 5. Build Plan for Attempt 1

The goal is a first successful link of `git` against `libnextglk.a` without
modifying any Git source files.

### Step 1 â€” Create `nextglk/Make.nextglk`

Analogous to CheapGlk's `Make.cheapglk`:

```makefile
LINKLIBS =
GLKLIB = -lnextglk
```

This is included by `git/Makefile` via:

```makefile
include $(GLKINCLUDEDIR)/$(GLKMAKEFILE)
```

where `GLK = nextglk`, `GLKINCLUDEDIR = ../nextglk`, `GLKMAKEFILE = Make.nextglk`.

### Step 2 â€” Create `nextglk/glkstart.h`

Provide the Unix startup interface that `git_unix.c` expects:

- `glkunix_argumentlist_t` struct typedef
- `glkunix_startup_t` struct typedef
- `glkunix_arguments[]` declaration (non-const, defined in glkstart.c)
- `glkunix_startup_code()` declaration
- `glkunix_stream_open_pathname()` declaration
- `glkunix_stream_open_pathname_gen()` declaration
- `glkunix_set_base_file()` declaration

### Step 3 â€” Create `nextglk/glkstart.c`

Provide implementations for:

- `glkunix_arguments[]` â€” command-line argument definition
- `glkunix_startup_code()` â€” parse args, call `nextglk_init()`, return 1
- `glkunix_stream_open_pathname()` â€” open file as Glk stream (returns NULL stub initially)
- `glkunix_stream_open_pathname_gen()` â€” open file with write mode option (stub)
- `glkunix_set_base_file()` â€” store base file path (stub)

### Step 4 â€” Add fileref stub functions

Add to a new file `nextglk/nextglk_fileref.c` (or to `nextglk.c`):

All 9 `glk_fileref_*` functions as stubs returning safe defaults (0 or NULL).

### Step 5 â€” Add missing image/graphics stubs

Add to `nextglk_image.c` or a new file:

- `glk_window_flow_break` â€” no-op
- `glk_window_erase_rect` â€” no-op
- `glk_window_fill_rect` â€” no-op
- `glk_window_set_background_color` â€” no-op

### Step 6 â€” Add remaining module stubs

Add to a new file `nextglk/nextglk_stubs.c`:

All remaining missing functions under their `#ifdef` guards:

- Unicode (16 functions, `#ifdef GLK_MODULE_UNICODE`)
- UnicodeNorm (2 functions, `#ifdef GLK_MODULE_UNICODE_NORM`)
- Sound (9 functions, `#ifdef GLK_MODULE_SOUND`)
- Sound2 (5 functions, `#ifdef GLK_MODULE_SOUND2`)
- Hyperlinks (4 functions, `#ifdef GLK_MODULE_HYPERLINKS`)
- DateTime (10 functions, `#ifdef GLK_MODULE_DATETIME`)
- Resource Stream (2 functions, `#ifdef GLK_MODULE_RESOURCE_STREAM`)

Add `gidispatch_set_retained_registry()` (no `#ifdef`, always needed).

### Step 7 â€” Modify `nextglk/Makefile`

Add new source files:

```makefile
OBJS = \
    ...existing... \
    glkstart.o \
    nextglk_fileref.o \
    nextglk_stubs.o
```

### Step 8 â€” Modify `nextglk/glk.h` (if needed)

Review `#ifdef` guards. If any module define prevents compilation of a stub
that references types/constants only defined inside the guard, the guard must
remain. Currently all modules are `#define`d, so all stubs will be compiled.

### Step 9 â€” Build and link

```bash
cd nextglk && make clean && make libnextglk.a
cd ../thirdparty/git && make clean && make GLK=nextglk
```

### Outcome

- `libnextglk.a` contains all ~90+ Glk API functions (most as stubs).
- `git` binary links successfully.
- `git` starts and calls `glkunix_startup_code()`, then `glk_main()`.
- `glk_main()` calls `git()` which calls `git_init_dispatch()`, then begins
  executing game bytecode.
- When the game calls Glk functions through the dispatch layer, most calls
  hit stub implementations.

## 6. What Will (and Won't) Work at Attempt 1 Completion

### Will work

| Capability | Why |
|------------|-----|
| VM boots | `git` links and enters `glk_main()` |
| Dispatch layer initialises | `gidispatch_set_*_registry` callbacks are wired |
| Window opens | `glk_window_open()` creates a single TextBuffer window |
| Text output | `glk_put_char/string/buffer` route to `nextglk_put_char()` |
| `glk_gestalt` queries | Version, CharOutput, Unicode return sensible values |
| `glk_exit()` | Terminates process |
| Event loop | `glk_select()` returns immediately with `evtype_None` |

### Will NOT work

| Gap | Why |
|-----|-----|
| Game file loading (non-MMAP) | `glkunix_stream_open_pathname()` returns NULL |
| Game file loading (MMAP) | `#ifdef USE_MMAP` path works without Glk library |
| Save/restore | All fileref functions are stubs |
| Keyboard input | All input request functions are stubs |
| Images | All image functions are stubs |
| Sound, Hyperlinks, DateTime | All are stubs |
| Unicode rendering | All Unicode output functions are stubs (returns 0) |

### Recommended MMAP path

Build Git with `USE_MMAP` defined. The MMAP path in `git_unix.c` does not call
`glkunix_stream_open_pathname()` â€” it opens the file directly with `open()`
and `mmap()`. This avoids the need for a working `glkunix_stream_open_pathname()`
in Attempt 1.

```makefile
OPTIONS = -DUSE_MMAP
```

With MMAP, the only glkstart symbols actually called at runtime are:
- `glkunix_argumentlist_t` / `glkunix_arguments[]` â€” data only
- `glkunix_startup_code()` â€” must return 1

The `glkunix_stream_open_pathname()` function is not called in the MMAP code
path. This means Attempt 1 can succeed with a minimal glkstart.c.
